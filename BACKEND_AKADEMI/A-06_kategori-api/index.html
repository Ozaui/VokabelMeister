A-06 — Kategori API (Categories) (tamamlandı)

Kelimeleri (WordConcept) kategoriye ayırmak: self-referencing hiyerarşi (`Category. ParentCategoryId`), A-05'teki WordConcept/Word ayrımıyla AYNI çoklu dil deseni (`CategoryTranslation`) ve dilden bağımsız bir M:N ilişki (`WordCategory`). Entity + EF Config + Migration'dan başlayıp Repository, MediatR Command/Query'ler, Controller, Validator ve testlere kadar A-05'in dikey dilim disipliniyle ilerledi. Son bölümde A-05'in "Category tabloları henüz yok" diye ertelediği iki not (`categories[]`, `categoryId` filtresi) kapatıldı — ve bu sırada A-05'ten kalma bağımsız bir audit-log hatası (deferred LINQ) da bulunup düzeltildi.

  1. 1 Neden Category + CategoryTranslation + WordCategory? Üç ayrı sorumluluk (hiyerarşi/çeviri/kelime bağı) neden üç ayrı tabloya bölündü, self-ref FK, WordConceptId (WordId değil) kararı.
  2. 2 Entity + EF Config Category/CategoryTranslation/WordCategory.cs, WordConcept.cs'e saf ekleme, CHECK constraint + self-ref FK Restrict + 12 kategori/24 çeviri HasData seed.
  3. 3 Migration + DbContext AddCategoriesSchema — 3 tablo tek migration'da, seed INSERT'leri, gerçek DB'ye uygulandı, DbContext'e 3 yeni DbSet.
  4. 4 Repository Katmanı — ICategoryRepository Silme koruması sorguları, WouldCreateCycleAsync (döngü koruması), CategoryDtoBuilder (neden AutoMapper değil, orphan promotion).
  5. 5 Command/Query Handler'ları — CRUD CreateCategoryCommand, UpdateCategoryCommand (döngü koruması), DeleteCategoryCommand (409), GetCategoriesQuery, GetCategoryWordsQuery, 3 exception.
  6. 6 Controller + Validator + Testler CategoriesController (5 endpoint) + 5 Postman örneği, CreateCategoryCommandValidator/UpdateCategoryCommandValidator, 21 birim testi.
  7. 7 Word Tarafının Retrofit'i (A-05'e Geri Dönüş) categories[]/categoryId — 8 dosyada kod-degisiklik, ARTI iki bağımsız hata düzeltmesi (deferred LINQ audit log hatası, tekrarlanan categoryId → 500 riski).
  8. 8 Özet + Sözlük A-06'nın tamamı: 3 entity, 5 Command/Query, 5 endpoint, 219/219 yeşil test.

A-07 — Admin API (Kullanıcı Yönetimi + İstatistik + Toplu Import + Log Görüntüleme) (tamamlandı)

A-05/A-06'nın aksine hiçbir yeni tablo AÇMIYOR — A-03'te zaten var olan `User` entity'sini admin gözünden yönetiyor: arama+rol filtreli liste, detay+istatistik, rol değiştirme, hesap dondurma/aktifleştirme, genel istatistik, toplu kelime import, log görüntüleme. TASK dosyasının A-06'dan sonra düzeltilen bir notuna göre ("IAdminService/AdminService" deseni terk edildi) baştan MediatR Command+Handler ile yazıldı. Bu görevin en önemli YENİ desenleri: rol/hesap durumu değişimi gibi hassas admin işlemleri projede İLK KEZ HEM `IActivityLogger` HEM `ISecurityLogger`'a birlikte yazıyor; log görüntüleme dilimi A-04'ten beri bekleyen `SecurityLog.Detail` Code-sonra-çöz borcunu kapatıyor. A-07 boyunca kod denetiminde İKİ gerçek düzeltme yapıldı: tüketicisiz yazılan `AdminStatisticsDto`/ `GetStatisticsAsync` geri alınıp gerçek tüketicisiyle yeniden eklendi, ve bir admin'in kendi rolünü/hesap durumunu değiştirebilmesi (self-lockout riski) kullanıcı onayıyla kapatıldı.

  1. 1 Neden Entity Yok + Repository Genişletmesi A-03'ün User entity'si üzerinden yönetim (yeni tablo/migration yok), "IAdminService" deseninin terki, IUserRepository.GetPagedAsync, kod denetiminde bulunup çıkarılan tüketicisiz GetStatisticsAsync/AdminStatisticsDto.
  2. 2 Command/Query Handler'ları — Liste + İkili Loglama GetUsersQuery (temsili query), UpdateUserRoleCommand (temsili command — IActivityLogger + ISecurityLogger birlikte, projede ilk örnek), GetUserByIdQuery/UpdateUserStatusCommand aynı pattern.
  3. 3 Controller + Validator + Testler AdminController (4 endpoint, controller-seviyesi tek [Authorize(Roles="Admin")]) + 4 Postman örneği, UpdateUserRoleCommandValidator, 4 test dosyası — 231/231 yeşil test. Ayrıca kod denetiminde bulunan self-lockout eksikliği: admin kendi rolünü/durumunu değiştiremez.
  4. 4 İstatistik Dilimi — GetAdminStatisticsQuery Bölüm 1'de geri alınan AdminStatisticsDto/GetStatisticsAsync'in gerçek tüketicisi, GetTotalCountAsync (Word/Category), bellekte kayıt grafiği gruplaması, bilinçli olarak yazılmayan LoginsByDay, GET /admin/statistics — 235/235 yeşil test.
  5. 5 Toplu Kelime Import — BulkImportWordsCommand Icerik.md "Eşleştirme" tasarımına göre her satır bağımsız tek-dilli WordConcept, A-05'in WordTranslationInput/WordEntityBuilder/WordGrammarValidator'ının yeniden kullanımı, best-effort (bir satır hatalıysa diğerleri durmaz), TEK BULK_IMPORT_WORDS ActivityLog kaydı, POST /admin/words/import — 239/239 yeşil test.
  6. 6 Log Görüntüleme — Code-Sonra-Çöz Deseninin İkinci Uygulaması LogMessages.cs (A-04'ten beri bekleyen SecurityLog.Detail çözme borcu kapandı), neden ActivityLog.Action/OldValue/NewValue kapsam dışı, GetActivityLogsQuery/GetApplicationLogsQuery/GetSecurityLogsQuery, 3 endpoint — 244/244 yeşil test.
  7. 7 Özet + Sözlük A-07'nin tamamı: 4 dilim, 9 endpoint, 9 yeni Command/Query, 244/244 yeşil test, en büyük ders — "önce yaz sonra kullanan kodu bekle" YAGNI ihlalidir.

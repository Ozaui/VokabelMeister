A-08 — Medya / Dosya Yükleme API (tamamlandı)

Admin panelin bir kelime kartına görsel ekleyebilmesi için gereken tek şey: bir yükleme uç noktası. A-05'te zaten var olan `WordConcept.ImageUrl` alanına yazılacak URL'i üreten `IFileStorageService`/`LocalFileStorageService` (uzantı/boyut doğrulama, benzersiz ad üretimi), projedeki İLK `multipart/form-data` uç noktası (`MediaController`, HealthController ile AYNI desende MediatR DIŞINDA), ve `UseStaticFiles` ile yüklenen dosyanın tarayıcıdan doğrudan görüntülenmesi. A-08'in en önemli tespiti bir KAPSAM düzeltmesi oldu: TASK dosyasının ilk hâli "Word.ImageUrl + migration" diyordu, ama bu alan A-05'te zaten yazılmıştı — yeni migration GEREKMEDİ.

  1. 1 Dosya Depolama Servisi — IFileStorageService Kapsam düzeltmesi (Word.ImageUrl A-05'te zaten vardı), IFileStorageService neden Application/Services'te (Infrastructure DEĞİL), IFormFile yerine saf Stream+metadata, LocalFileStorageService (uzantı/boyut doğrulama, Guid tabanlı benzersiz ad), UnsupportedFileTypeException/FileTooLargeException.
  2. 2 MediaController + UseStaticFiles HealthController ile aynı desen (MediatR dışında), projedeki ilk multipart/form-data uç noktası, POST /media/images/upload Postman örneği, IActivityLogger (UPLOAD_MEDIA, EntityId=NULL), app.UseStaticFiles() (Program.cs).
  3. 3 Testler + Özet + Sözlük FileStorageServiceTests (Moq yerine gerçek geçici klasör, 6 test), A-08'in tamamı: 250/250 yeşil test.

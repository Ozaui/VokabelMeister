Web Yol Haritası

Kullanıcı Web App'inin (/web — ayrı bir React+Vite projesi, Admin/Mobil ile kod paylaşmaz) her feature'ının (sayfa/component/hook) gerçek kodunun hangi sırayla ve neden öyle yazıldığını adım adım gösteren junior eğitim rehberi.

Altın kural: Bir feature'ı tüm katmanlarıyla bitir, sonra diğerine geç — katman katman değil, feature feature. Her katmanı yazar yazmaz hemen (1) ilgili TASK/D_web_app.md dosyasında işaretle, (2) o feature'ın HTML sayfasına kod + açıklamayla ekle. Feature bitince toplu yazmak yasak.
♻️ Yeniden kullanılan kod: Bir feature daha önce yazılmış bir component/hook/ slice'ı (örn. useAuth, ortak Button) kullanıyorsa, o kodun tam hâli o feature'ın sayfasına da yazılır — her feature tek başına baştan sona okunabilir olmalı.

Bir Feature'ın Yazım Sırası

  1. TS Tipi/Arayüz — API'dan gelen/giden verinin şekli
  2. ➜ Bu kodu HTML sayfasına işle
  3. RTK Query Endpoint — API'a istek atan query/mutation
  4. ➜ Bu kodu HTML sayfasına işle
  5. Redux Slice — yalnızca ek local/UI state gerekiyorsa
  6. ➜ Bu kodu HTML sayfasına işle
  7. Custom Hook — varsa, iş mantığını component'ten ayırır
  8. ➜ Bu kodu HTML sayfasına işle
  9. Component — JSX + mantık, yukarıdakileri birleştirir
  10. ➜ Bu kodu HTML sayfasına işle
  11. Route/Import Entegrasyonu — nereye/nasıl import edildi
  12. ➜ Bu kodu HTML sayfasına işle
  13. Birim Test — RTL ile component/hook/slice testi
  14. ➜ Bu feature'ın HTML sayfasına işle
Yeni feature eklemek: _TASLAK.html'i kopyala → <faz>_<id>.html yap → FEATURE objesini doldur → aşağıdaki LISTE'ye bir satır ekle. Detay → ../TASK.md (⭐ Frontend Çalışma Yöntemi).

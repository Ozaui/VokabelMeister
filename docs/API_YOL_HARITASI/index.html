API Yol Haritası

Her API'ın gerçek kodunun hangi sırayla ve neden öyle yazıldığını adım adım gösteren junior eğitim rehberi. Her API'ın kendi sayfası var; aşağıdan açın.

Altın kural: Bir API'ı tüm katmanlarıyla bitir, sonra diğerine geç — katman katman değil, API API. Her katmanı yazar yazmaz hemen (1) TASK.md'de işaretle, (2) o API'ın HTML sayfasına kod + açıklamayla ekle. API bitince toplu yazmak yasak.
♻️ Yeniden kullanılan kod: Bir API daha önce yazılmış bir kodu/yardımcıyı (örn. Repository<T>, PasswordService.Hash, ortak DTO) kullanıyorsa, o kodun tam hâli o API'ın sayfasına da yazılır — her API tek başına baştan sona okunabilir olmalı.

Bir API'ın Yazım Sırası

  1. Entity — veri modeli
  2. ➜ Bu kodu HTML sayfasına işle
  3. EF Konfigürasyon — FK, index, check, soft delete
  4. ➜ Bu kodu HTML sayfasına işle
  5. Migration — tabloyu DB'ye ekle
  6. ➜ Bu kodu HTML sayfasına işle
  7. Request DTO + Validator
  8. ➜ Bu kodu HTML sayfasına işle
  9. Exception(lar) — varsa
  10. ➜ Bu kodu HTML sayfasına işle
  11. Repository (arayüz + impl)
  12. ➜ Bu kodu HTML sayfasına işle
  13. Response DTO
  14. ➜ Bu kodu HTML sayfasına işle
  15. Servis (arayüz + impl)
  16. ➜ Bu kodu HTML sayfasına işle
  17. Controller
  18. ➜ Bu kodu HTML sayfasına işle
  19. DI kaydı
  20. ➜ Bu API'ın HTML sayfasına işle
Yeni API eklemek: _TASLAK.html'i kopyala → <faz>_<id>.html yap → API objesini doldur → aşağıdaki LISTE'ye bir satır ekle.
